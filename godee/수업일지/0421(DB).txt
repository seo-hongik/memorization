데이터 사전


사용자와 데이터베이스 자원을 효율적으로 관리하기 위한 다양한 정보를 저장하는 시스템 테이블의 집합이다 . 

사용자가 테이블을 생성하거나 사용자를 변경하는 등의 작업을 할 때 데이터 베이스 서버에 의해 자동으로 갱신되는 테이블로,

사용자는 데이터 사전의 내용을 직접 수정하거나 삭제할 수 없고 

사용자가 이해할 수 있는 테이터를 산출해 줄 수 있도록 하기 위해서 읽기 전용 뷰 형태로 정보를 제공합니다. 

-------------------
데이터 사전 종류 

USER_ 데이터 사전 

자신의 계정이 소유한 객체 들에 관한 정보 조회 

접두어로 USER가 붙은 데이터 사전은 사용자와 가장 밀접하게 관련된 뷰로서 자신이 생성한 테이블 인덱스/ 뷰, 동의어 등의 객체나 
해당 사용자에게 부여된 권한 정보를 제공합니다. 

USER_SEQUENCES는 사용자가 소유한 시퀀스의 정보를 조회할 수 있는 데이터 사전이고 
USER_INDEXES는 사용자가 소유한 인덱스 정보를 조회할 수 있는 데이터 사전이고, 
USER_VIEWS는 사용자가 소유한 뷰정보를 조회할 수 있는 데이터 사전입니다. 

데이터 사전은 USER_뒤에 원하는 객체 등을 기술해 주면되고 
뒤에 기술되는 몇칭은 일반적으로 's'가 붙는 복수 타입임을 주의합니다.  

---------------------

ALL_ 데이터 사전

자신 계정 소유 또는 권한을 부여 받은 객체 등에 관한 정보 조회

접두어로 ALL이 붙은 데이터베이스는 전체 사용자와 관련된 뷰로서 사용자가
접근 할 수 있는 모든 객체에 대한 정보를 조회할 수 
있습니다. 

----------------------
DBA_ 데이터 사전

데이터베이스 관리자만 접근 가능한 객체 등의 정보 조회 

접두어로 DBA가 붙은 데이터 사전은 시스템 관리와 관련된 
뷰 입니다.

CH09 데이터 조작과 트랜잭션 //DML(Data Manipulation Language)

테이블에 내용을 추가하는 INSERT문

데이터 조작어(DML)는 테이블에 새로운 데이터를 삽입 하거나 
기존의 데이터를 수정 하거나 삭제하기 위한 명령어 집합니다. 

테이블에 데이터를 입력하기 위한 명력어 입니다. 

INSERT INTO  tables_name
(column_name1 ,....,colums_nameN)
VALUES(column_name1 ,....,colums_valueN);

------------------------------

INSERT문

한번에 하나의 로우만 INTO 다음에 명시한 테이블에 삽입합니다. 

테이블명 다음에 기술한 칼럼에 VALUES 절에서 지정한 컬럼값을 순서대로 입력합니다. 

INTO 절에 컬럼을 명시하지 않으면 테이블을 생성할 때 정의한 칼럼 순서와 동일한 순서대로 VALUES이하읜 값이 입력되기 때문에 

테이블에 존재하는 모든 컬럼에 대해서 값을 모두 지정해야 합니다. 

입력되는 데이터 타입은 컬럼의 데이터 타입과 동일 해야 하며 
입력 되는 데이터 크기는 컬럼의 크기보다 작거나 동일해야 합니다. 

******삽입할 컬럼의 데이터 타입이 문자와 날짜일 경우에는 반드시 작은 따옴표('  '),를 사용해야 합을 유의바랍니다. 
-------------------------------
NULL 값을 갖는 로우 삽입

데이터를 입력하는 시점에서 해당 컬럼 값을 모르거나 확정되지 않았을 경우에는 컬럼의 값으로 NULL을 입력합니다.

--테이블의 내용을 수정하는 UPDATE 문

테이블에 저장된 데이터를 수정하기 위한 DML입니다. 

WHERE 절을 생략하면 테이블에 있는 모든 행이 수정됩니다. 

UPDATE table_name1
SET   colunmn_name=value1, ....,colunmn_nameN=value1N
WHERE  conditions;


-----------------------

테이블의 내용을 삭제하는 DELETE 문

테이블에서 기존에 저장 되어 있던 데이터를 삭제할 수 있습니다. 

DELETE [FROM] table_name
WHERE  conditions;
