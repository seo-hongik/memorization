<!DOCTYPE html>

웹 브라우저가 현재 웹페이지가 html5 문서임을 인식하게 만들어 줍니다.

페이지가 작성된 html 버전에 대해 웹 브라우저에 지시합니다. 

<!DOCTYPE> 선언은 대소 문자 구분하지 않습니다. 


<!DOCTYPE> 선언은 html문서에서 <html> 태그 앞에 가장 먼저 선언되어야 합니다. 

------------------

<html lang="ko">

lang 속성은 요소 내용의 언어를 지정합니다. 

-----------------

<meta> 태그(요소)

메타데이터는 데이터에 대한 데이터(메타 정보)입니다. 

ex> 시계- 시계의 분침의 재질

<meta>태그는 html문서에 대한 메타 데이터를 제공합니다. 

메타 데이터는 페이지에 표시되지 않지만 컴퓨터 구분 분석이 가능합니다.

메타 데이터는 해드 사이에 있어야한다. 

-------------
charset 속성

charset 속성 html 문서의 문자 인코딩을 지정합니다. 

UTF-8 : 유니 코드 문자 인코딩 (중요)

ISO-8859-1 : 라틴 알파벳 문자 인코딩 (중요하지 않음)

EUC-KR : 한글 문자 인코딩

-----------------


name 속성

name 속성은 메타 데이터의 이름을 지정합니다. 

application -name : 페이지가 나타내는 웹 응용 프로그램의 이름을 지정합니다. 

author : 문서 작성자의 이름을 지정합니다. 

description : 페이지의 설명을 지정합니다. 검색엔진은 이 설명을 선택하여 검색 결과와 함께 표시 할 수 있습니다. 

generator :문서를 생성하는 데 사용되는 소프트웨어 페키지 중 하나를 지정합니다. 

keywords : 페이즈와 관련된 쉼표로 구분된 키워드 목록을 지정합니다. 

viewport : 장치에 따라 다르며 컴퓨터 화면보다 휴대 전화에서 더 작습니다. 

	뷰포트(웹 페이지의 사용자가 볼 수 있는 영역)을 조정합니다. 


subject : 문서 제목 정보

robots : 검색 로봇 제어

copyright : 저작권 정보

reply-to : 연락처 (이메일)

date : 날짜 (작성일)

--------------------------

검색 로봇 제어

일부 페이지를 검색로봇의 수집대상에서 제외하고 싶은 경우도
있습니다. 검색 로봇에 대한 선언은 <meta name="robots">라는 형식으로 지정합니다. 

content 속성에는 다음과 같은 값을 지정할 수 있습니다. 

	복수로 지정할 때에는 콤마로 구분합니다. 

	All(기본값) : 'INDEX,FOLLOOW'를 지정한 것과 같습니다. 

	NONE : 'NOINDEX,NOFOLLOW'를 지정한 것과 같습니다. 

	INDEX : 그 페이지를 수집대상으로 합니다. (검색 결과에 나타납니다.)

	FOLLOW : 그페이지를 포합해 링크가 걸린 곳을 수집대상으로 합니다. 

	NOINDEX : 그 페이지를 수집대상에서 제외합니다. (검색 겨로가에 나타나지 않습니다.)

	NOFOLLOW : 그 페이지를 포함해 링크가 걸린 곳을 수집대상으로 하지 않습니다.

구조언어(HTML)

(HTML은 'HyperText Markup Language' 의 약자로 웹 문서의 구조론을 정의할 때 사용합니다. 
제목 본문 목록 링크 이미지 등의 다양한 콘텐츠를 의미있게 마크업할 수 있습니다. 
---------------------

구조언어 (XML)

XML (eXtensible Markup Language)은 1996년 W3C에서 제안한 것으로 
웹에서 구조화된 문서를 전송할 수 있도록 설계된 표준화된 텍스트 형식입니다. 

XML은 인터넷에서 기존에 사용하던 HTML의 한계를 극복하고 
SGML의 복잡함을 해결하는 방안으로 HTML에 담겨져 있는 형식적 요소를 
완전히 배제하는 방식입니다. 

----------------------

XHML (eXtensible HyperText Markup Language)
을 'XML 응용으로서의 HTML4를 다시 공식화 한것' 이라고 정의하고 있습니다.

W3C는 좀더 원할하게 기계적으로 처리하도록 XML의 형식을 
빌어 HTML4.01를 재정의 하게 되는 것이 바로 XHTML입니다.

----------------------

HTML5 


HTML5는 HTML의 다음 버전으로 HTML4를 업그레이드한 것입니다. 
HTML5는 특정 플러그인에 의존하지 않고 콘텐츠를 제공하는 것이 목표입니다. 


---------------------


표현언어(CSS)

CSS(Cascading Style Sheet)는 HTML3.2부터 지원하기 시작된 것으로 
웹 디자인너와 자용자들의 필요에 의해 특별히 개발 되었습니다. 

CSS에서는 폰트, 색상, 공백 , 공간과 그밖의 문서 표현 측면을 자유롭게 지정할 수 있는 기능을 제공합니다. 

기본의 HTML은 웹 문서를 다양하게 설계하고 수시로 변경하는 데 많은 제약이 있었기 때문에 이것을 보완하기 위해 
CSS를 만든 것 입니다. 


-------------------

동작 및 제어 언어(DOM & ECMA Script)
웹 페이지의 요소를 객체화해서 동작을 제어하기 위해 사용하는 웹표준 기술에는 DOM과 ECMA Script가 있습니다. 

DOM(Document Obejct Model)은 웹페이지의 구성 체계를 말하는데 
이것은 HTML을 작성하면서 생성되는 논리적 규칙입니다. 다시 말해서 별도로 저작자가 구성하는 것은 아닙니다. 
웹브라우저는 이 논리적 구성체계인 DOM을 해석하여 페이지를 표시하거나 EMCA Script 등의 기술을 통하여 DOM의 구조를 변경할 수 있습니다.
