트랜잭션 관리 

데이터의 일관성을 보장합니다. 

트랜잭션(Transaction)은 데이터 처리에서 논리적으로 하나의 작업 단위를 의미합니다. 

오라클은 하나의 트랜잭션 역시 ALL-OR-NOTHING 방식으로 처리합니다.

DML작업이 성공적으로 처리하도록 하기 위해서 COMMIT 명령을, 작업의 취소하기 위해서는 ROLLBACK 을 실행 합니다. 

COMMIT 모든 작업들을 정상적으로 처리하겠다고 확정하는 명령어로 트랜잭션의 처리 과정을 데이터베이스에 반영하기 위해
변경되 내용을 모두 영구 저장힙니다. 

COMMIT 명령어를 수행하게 되면 하나의 트랜잭션 과정을 종료하게 됩니다. 

ROLLBACK 작업 중 문제가 발생하여서 트랜잭션의 처리 과정에서 발생한 변경 사항을 취소하는 명령어 입니다. 

ROLLBACK 명령어 역시 트랜잭션 과정을 종료하게 됩니다. 

ROLLBACK 은 트랜잭션으로 인한 하나의 묶을 처리가 시작되기 이전의 상태로 되돌립니다. 

------------------
트랜잭션의 특징 (ACID )

원자성 (Atomicity) : 모든 작업이 반영되거나 모두 롤백되는 특성입니다. 

*일관성(Consistency): 데이터는 미리 정의된 규칙에서만 수정이 가능한 특성을 의미합니다. 숫자컬럼에 문자열 값을 저장이 안되도록 
보장해줍니다. 

고립성 (독립성, isolation) : A와 B두개의 트랜잭션이 실행이 되고 있을 때 A의 작업들이 B에게 보여지는 정도를 의미합니다. 

*영구성 (지속성, DUrability) : 한번 반영(커밋)된 트랙잭션의 내용은 영원히 적용되는 특성을 의미합니다.

-------------------------

트랜잭션 보충 설명 트랜잭션은 첫번째 DML문이 실행될 때 시작되어 다음 이벤트가 발생하면 종료됩니다. 

즉, COMMIT 혹은 ROLLBACK 문이 실행 되는 경우 DDL 혹은 DCL 문이 실행 되는 경우 오라클이 종료 되는 경우입니다. 

트랜잭션이 종료되면 실행가능한 다음 SQL 문이 다음 트랜잭션을 자동으로 시작합니다. 
DDL 혹은 DCL 문이 실행 되는 자동으로 COMMIT되고 오라클이 종료되면 자동으로 ROLLBACK 됩니다. 
-------------------------

데이터 무결성 제약 조건

NOT NULL : 컬럼에 NULL값을 포함하지 못하도록 지정합니다. 

UNQUE : 테이블의 모든 로우에 대해서 유일한 값을 갖도록 합니다. 

PRIMARY KEY 테이블의 각 행을 식별하기 위한 것으로 NULL과 중복된 값을 모두 허용하지 않습니다. 단, NOT NULL조건과 

UNIQUE 조건을 결합 한 형태입니다. 


FOREIGN KEY: 참조되는 테이블에 컬럼 값이 항상 존재해야 합니다. 

CHECK: 저장 가능한 데이터 값의 범위나 조건을 지정하여 설정한 값만을 허용합니다. 

--------------------------

NOT NULL 제약조건

NOT NULL 제약조건을 사용하면 해당 컬럼이 NULL값을 가질 수 없게 됩니다. 

NOT NULL 제약조건은 컬럼 레벨로만 정의할 수 있습니다. 

--------------------------

유일한 값만 허용하는 UNIQUE 제약조건

UNIQUE 제약 조건은 특정 컬럼에 모든값이 고유하게 유지되도록 하는 고유키를 생성합니다. 

NULL은 UNIQUE 제약 조건에 위반되지 않으므로 NULL값을 허용합니다.

column_name data_type CONSTRAINT constraint_neme
constraint_type
