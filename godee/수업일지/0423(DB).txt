데이터 구분을 위한 PRIMARY KEY 제약조건

테이블의 기본키를 생성합니다. 

PRIMARY KEY 제약 조건을 지정한 컬럼은 중복된 데이터를 저장하지 못하도록 제약하기 때문에 컬럼에 고유성을 부여합니다. 

또한 기본키에 속하는 컬럼은 NULL값을 가질 수 없도록 합니다. 

-----------------

참조 무결성 위한 FORINGN KEY제약 조건

테이블에 외래키 제약 조건을 생성합니다. 

외래키 즉 참조 무결성은 테이블과 테이블 사이의 주종 관계

설정을 위한 제약 조건입니다. 

외래키는 부모 테이블의 컬럼 값을 참조하는 자식 테이블의 칼럼 입니다. 

부모키(참조키)는 자식테이블에서 참조하는 부모 테일블의 컬럽입니다. 부모키가 되기 윈한 컬럼은 반드시 부모테일블의 기본키나 
유일키로 설정되어 있어야 합니다. 


--------------------
값의 범위를 설정하는 CHECK 제약 조건

컬럼에서 허용 가능한 데이터의 범위나 조건을 정의합니다. 

*CURRVAL, NEXTVAL, ROWNUM과 같은 의사 칼럼이나 SYSDATE, USER와 같은 함수에는 사용할 수 없습니다. 

--------------------

DEFAULT 제약조건

디폴트는 아무런 값을 입력하지 않았을 때 디폴트 제약의 값이 입력 됩니다. 

--------------------

제약 조건 변경하기 

제약 조건 추가 

ALTER TABLE  teble_name 

ADD [ONSTRAINT constraint_name] constriant_type (column _ name);


NULL 무결성 제약 조건은 ALTER..ADD 명령문으로 추가하지 못합니다. 

ALTER TABLE ...MODIFY 명령문을 사용하면  NULL 상태로 변경하는 것은 가능합니다. 

--------------------

--NULL 값 허용하지 않기 
ALTER TABLE emp_copy
MODIFY ename CONSTRAINT emp_copy_ename_nn NOT NULL;

--------------------
제약 조건 제거하기

ALTER_TABLE table name
DROP PRIMARY KEY | UNIQUE (clumn) | CONSTRAINT
constraint_name[CASCADE];

외래키 제약조건으로 지정되어 있는 부모테이블의 기본키 제약조건을 제거하려면 
자식 테이블의 참조 무결성 제약 조건을 먼저 제거한 후 제거하거나
CASCADE 옵션을 사용합니다.

------------------
제약 조건 비활성화

ALTER TABLE 명령 문에 DISABLE 절을 사용하면 제약 조건을 삭제하지 않고 일시적으로 비활성화 합니다. 

ALTER TABLE  	table_name
DISABLE CONSTRAINT constraints_name[CASCADE];

------------------

제약 조건 활성화


ALTER TABLE  	table_name
ENABLE 	CONSTRAINT constraints_name[CASCADE];
