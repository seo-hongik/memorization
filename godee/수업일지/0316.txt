-------------------

universal design

모든 사람을 위한 디자인 (Design for all people)
이라고도 하며 
미국 로널드 메이스에 의해 처음 이야기 되었습니다. 

장애의 유무나 연령 등에 관계없이 모든 사람들이 제품, 건축, 환경 , 서비스 등을 보다 편하고 안전하게 이용할 수 있도록 설계
하는 것을 말합니다. 

예로 물건을 쥐는 힘이 약한 사람들을 위해 레버식 문 손잡이 설치나 휠체어가 이동할 수 있도록 계단 대신 경사로를 설치하는 것을 들 수 있습니다. 


-----
웹(www) 서비스의 이해 

client ->요청(request) Server
       <-응답(respense)

웹의 동장 방식은 서버(server)와 클라이언트(client)의 형태를 
띄며 클라이언트가 웹 브라우저를 통해 특정 콘첸츠를 요철하면 서버는 클라이언트의 요청에 응답하는 형식으로 이루어져 있습니다.

------------

웹프로그래밍의 이해 

클라이언트 사이드 : HTML, CSS, DOM, ECAM Script, XML등

서버사이드 : JSP, ASP, PHP

웹 프로그램은 수행되는 위치에 따라 클라이언트측과 서버측의로 분류할 수 있으며, 컴파인 여부에 따라 '컴파일방식'과 비컴파일방식(Script 포함_)으로 나눌 수 있습니다. 

클라이언트가 요청한 파일이 서버의 우선처리 없이 클라이언트에 도착한 후 웹 브라우저로 수행되면 '클리이언트 사이드'로 분류하고 서버에서 우선 처리한 후 클라이언트에서
수행 되면 '서버 사이드 '로 분류합니다.

-------------

시맨틱 웹

웹의 미래는 '참여', '공유', '개방'을 의미 하는 웹 2.0에서 '개인화' '지능화'의 웹 3.0d으로 변화하고 있습니다. 

그렇다면 웹 3.0이 의미하는 지능화는 무엇을 뜻할까요?
	-> 바로 차세대 지능형 웹인 시맨틱웹(semantic web)을 말합니다. 

시맨틱 웹은 웹 표현 방식인 html의 한꼐를 극복하기 위해 등장한 xml언어를 기반하으로 하는 차세대 웹으로 , 시맨틱 웹이 구현하려는 환경은 
인간의 언어구조가 기계에게도 이해 될 수 있도록 하는 것입니다. 

시맨틱 웹은 현존하는 웹(www)을 대체하는 개념이 아니라 좀더 체계화하는 데에 목적이 있습니다.

<br> 태그

<br> 태그는 한 줄 바꿈을 삽입합니다. 

<br>태그는 빈 태그이므로 종료(닫는) 태그가 없습니다.

<address> 태그 


<adress> 태그는 문서 또는 기사 작성자 / 소유자의 연락처 
정보를 정의합니다. 


<adress> 요소가 <body> 요소 안에 있으면 문서의 연락처 정보를 나타냅니다. 

<address> 요소가 <article> 요소 안에 있으면 해당 기사의 연락처 정보를 나타냅니다. 

<adress> 요소의 텍스트는 일반적으로 기울임 글꼴로 렌터링 됩니다. 대부분 브라우저는 주소 요소 앞뒤에 줄 바꿈을 추가합니다.

<hr> 태그

<hr>태그는 HTML 페이지에서 주제별 구분 (예 :주제 이동)을 정의합니다. 

<hr> 요소는 HTML 페이지에서 컨텐츠를 분리하거나 변경을 정의하는 데 사용됩니다. 

XHTML에서는 <hr> 태그는 수평 규칙을 나타냅니다. 

그러나 <hr> 태그는 여전히 시각적 브라우저에서 가로 규칙으로 표시될 수 있지만 이제는 표현 용어가 아닌 의미적 용어로 정의됩니다.

Ch03 링크와 이미지

<a> 태그

<a> 태그는 한 페이지에서 다른 페이지로 연결하는 데 사용되는 하이퍼 링크를 정의합니다. 

<a> 요소의 가장 중요한 속성은 링크의 대상을 나타내는 href속성 입니다. 

기본적으로 링크는 모든 브라우저에서 다음과 같이 나타납니다. 
	방문하지 않은 링크는 밑줄과 파란색으로 표시됩니다. 
	방문한 링크는 밑줄이 그어져 있으며 자주색으로 표시됩니다. 
	활성화된 링크는 밑줄과 빨간색으로 표시됩니다. 


href 속성 : 링크가 가는 페이지의 URL 지정해줍니다.  <a href="http:/www.google.co.kr">구글</a>
Target 속성 : 연결된 문서를 열 위치를 지정합니다. <a href="http://www.naver.com" target"_blank">네이버</a>

Ch06 테이블 관련 요소

---------------------

<table> 태그는 HTML 테이블을 정의합니다. 
HTML 태그는 <table> 요소와 하나 이상의 <tr>,<th> 및 <td> 요소로 구성됩니다. 

<tr>요소는 테이블 행을 정의하고 <th> 요소는 테이블 헤더를 정의하며, <td> 요소는 테이블 셀을 정의합니다. 

<caption> 태그는 테이블 캡션을 정의합니다. 

<table> 태그 바로 뒤에 <caption> 태그를 삽입해야 합니다. 



-----------------

<thead> 태그

<thead> 태그는 HTML테이블에서 헤더 내용을 그룹화 하는데 사용합니다. 

<thead> 요소는 <tbody> 및 <tfoot> 요소와 함께 사용되어 테이블의 각 부분(머리글, 본문, 바닥글)을 지정합니다. 

브라우저는 이 요소를 사용하여 머리글 및 바닥글과 독점적으로 테이블 본문을 스크롤 할 수 있습니다. 또한 여러 페이지에 
걸쳐있는 큰 테이블을 인쇄할 때 이러한 요소를 사용하면 각 페이지의 맨 위와 맨 아래에 테이블 머리글과 바닥글을 인쇄할 수 있습니다. 

<thead> 태그는 다음 컨텍스트를 사용합니다. <table> 요소의 하위 요소 뒤 및 <tbody>, <tfoot> 및 <tr> 요소 앞

<thead> 요소에는 하나 이상의 <tr> 태그가 있어야 합니다. 

----------

<tbody> 태그

<tbody> 태그는 HTML 테이블에서 본문 내용을 그룹화 하는데 사용합니다. 

<thead> 요소는 <tbody> 및 <tfoot> 요소와 함께 사용되어 테이블의 각 부분(머리글, 본문, 바닥글)을 지정합니다. 

브라우저는 이 요소를 사용하여 머리글 및 바닥글과 독점적으로 테이블 본문을 스크롤 할 수 있습니다. 또한 여러 페이지에 
걸쳐있는 큰 테이블을 인쇄할 때 이러한 요소를 사용하면 각 페이지의 맨 위와 맨 아래에 테이블 머리글과 바닥글을 인쇄할 수 있습니다. 

<tbody> 태그는 <caption> <colgroup> 및 <thead> 요소 다음에 <table> 요소의 자식으로 사용되어야 합니다. 


--------------


<tfoot> 태그

<tfoot> 태그는 HTML 테이블에서 바닥글 내용을 그룹화 하는 데 사용합니다.

<tfoot> 요소는 <thead> 및 <tbody> 요소와 함께 사용되어 테이블의 각 부분(머리글, 본문, 바닥글)을 지정합니다. 

브라우저는 이 요소를 사용하여 머리글 및 바닥글과 독점적으로 테이블 본문을 스크롤 할 수 있습니다. 또한 여러 페이지에 
걸쳐있는 큰 테이블을 인쇄할 때 이러한 요소를 사용하면 각 페이지의 맨 위와 맨 아래에 테이블 머리글과 바닥글을 인쇄할 수 있습니다. 

<tfoot> 태그는 <caption> <colgroup> 및 <thead> 요소 다음에 <table> 요소의 자식으로 사용되어야 합니다. 
-------------

<th> 태그

<th> 태그는 HTML 테이블에서 헤더 셀을 지정합니다. 

HTML 테이블에는 두 가지 종류의 셀이 있습니다. 

	헤더 셀 : 헤더 정보를 포함합니다. (<th> 요소로 작성됨)
	표준 셀 : 데이터를 포함합니다(<td> 요소로 작성됨)
<th> 요소의 텍스트는 기본적으로 굵게 표시되고 가운데에 표시됩니다. 

<td>요소의 텍스트는 기본적으로 규칙적이고 왼쪽 정렬 됩니다.
------------------
<tr> 태그

<tr> 태그는 HTML 테이블에서 행을 정의합니다. 

<tr> 요소는 하나 이상의 <th> 또는 <td> 요소를 포함합니다. 

---------------------------

<td> 태그는 HTML 테이블에서 표준 셀을 정의합니다. 

헤더 셀 : 헤더 정보를 포함합니다. (<th> 요소로 작성됨)
	표준 셀 : 데이터를 포함합니다(<td> 요소로 작성됨)
<th> 요소의 텍스트는 기본적으로 굵게 표시되고 가운데에 표시됩니다. 

<td>요소의 텍스트는 기본적으로 규칙적이고 왼쪽 정렬 됩니다.

colspan 속성 : 셀이 확장해야 하는 열의 수를 지정합니다. 
rowspan 속성 : 셀이 확장해야 하는 행의 수를 지정합니다. 
----------------------

테이블 핵심 정리

<tr> 태그는 table row를 의미합니다. 
<th> 태그는 table head를 의미합니다.
<td> 태그는 table data를 의미합니다. 

border 속성 : 표의 테두리 두께를 지정합니다.
